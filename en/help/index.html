init

Initializes a website. If folder isn’t defined, Hexo will set up the website in the current directory.

new

Creates a new article. If layout isn’t defined, it’ll equal the default_layout setting in _config.yml. If the title is more than one word, wrap it with quotation marks.

Alias: n

generate

Generates static files.

Alias: g

Option Description
-d, --deploy Deploy after generate done
-w, --watch Watch file changes

publish

Publishes a draft.

Alias: p

server

Starts server.

Alias: s

Option Description
-p, --port Override default port
-s, --static Only serve static files
-l, --log Enable logger. Override logger format.
-d, --drafts Serve drafts as normal posts.

deploy

Deploys your website.

Alias: d

Option Description
--setup Setup without deployment
-g, --generate Generate before deployment
-m, --message Customize commit message

render

Renders files.

Option Description
-o, --output Output destination

migrate

Migrates content from other blog systems.

clean

Cleans the cache file (db.json) and generated files (public).

list

Lists all routes.

version

Displays version information.

Options

Safe mode

Plugins and scripts won’t be loaded in safe mode. You can try this when you encounter some problems after installing a new plugin.

Debug mode

Displays verbose messages in terminal and saves log in debug.log. When you get some problems, try to run Hexo again in debug mode and please submit any errors to GitHub.

Silent mode

Hides output to the console.

Customize config file path

Customize config file path instead of _config.yml.
